7 ## Summary
8 
9 This is a C++ library for the Arduino IDE that allows you to manipulate general-purpose I/O (GPIO) pins on your Arduino or Arduino-compatible board with minimal overhead. Many function calls to this library will be inlined and compiled down to just one or two assembly instructions.
10 
11 This library is intended to be a replacement for the default Arduino I/O functions (`pinMode`, `digitalWrite`, and `digitalRead`) and has several advantages over them:
12 
13 * The compiled code runs faster.
14 * Operates only on the specific bits used by the pin instead of doing read-write-modify operations on entire registers.
15 * Never disables interrupts.
16 * Supports reading and restoring the state of a pin, which is useful if the pin is used for multiple purposes.
17 * Pins can be specified either by their AVR pin name or their Arduino pin name.
18 * Supports pins PD5 and PE2 on the ATmega32U4; these pins do not have an official Arduino pin number so they cannot be used with the default Arduino I/O functions.
19 
20 To use this library, you will need a basic understanding of microcontroller I/O pins. General information on this topic can be found in the Arduino [Digital Pins tutorial](http://arduino.cc/en/Tutorial/DigitalPins).
21 
22 Currently, this library requires that the pin number you specify must be known at compile time. This means the pin number cannot come from a variable that might change and it cannot be the result of a complicated calculation. This allows the compiler to generate optimal code for manipulating that pin.
23 
24 The Arduino GPIO functions automatically turn off a pin's PWM function when accessing it with `digitalRead` or `digitalWrite`. This library does not have that feature.
25 
26 ## Supported platforms
27 
28 This library is designed to work with the Arduino IDE versions 1.0.x, 1.5.x, and 1.6.x, and will probably not work with earlier versions.
29 
30 This library currently supports any board based on the ATmega32U4 or ATmega328P.

32 ## Getting started
33 
34 Download the [fastgpio-arduino library from GitHub](https://github.com/pololu/fastgpio-arduino/), decompress it, and drag the "FastGPIO" folder into the "libraries" subdirectory inside your Arduino sketchbook directory. You can view your sketchbook location by selecting File->Preferences in the Arduino environment. If there is not already a "libraries" folder in that location, you should create it yourself. After installing the library, restart the Arduino environment. Example code for using this library can be found in the File->Examples menu.

36 ## Specifying a pin
37 
38 There are three ways to specify a pin:
39 * By its pin number
40 * By its AVR pin name, using the `IO_*` macros that this library provides. For example, `IO_D3` refers to pin PD3.
41 * By alternative names defined in the Arduino IDE's `pins_arduino.h` header, such as `A2` and `MOSI`.
42 * By the special `IO_NONE` macro. Commands operating on the `IO_NONE` pin will have no effect.
43 
44 Any pin number that works with the standard Arduino I/O functions will work with the FastGPIO library.
45 
46 For a full list of all the supported pins, see the "Pin number reference" section below.

56 ## Digital inputs
57 
58 To configure a pin as a digital input with the internal pull-up resistor disabled, run:
59 
60 ~~~{.cpp}
61 FastGPIO::Pin<12>::setInput();
62 ~~~
63 
64 To configure a pin as a digital input with the internal pull-up resistor enabled, run:
65 
66 ~~~{.cpp}
67 FastGPIO::Pin<12>::setInputPulledUp();
68 ~~~
69 
70 Then, to read the value of the input, you can use:
71 
72 ~~~{.cpp}
73 bool inputHigh = FastGPIO::Pin<12>::isInputHigh()
74 ~~~

137 ## Speed test
138 
139 We compiled the following code with the Arduino IDE 1.5.8 and ran it on an ATmega32U4 running at 16 MHz:
140 
141 ~~~{.cpp}
142 #include <FastGPIO.h>
143 #include <util/delay.h>
144 
145 void setup() {
146  FastGPIO::Pin<A1>::setOutputLow();
147 }
148 
149 void loop() {
150  FastGPIO::Pin<A1>::setOutputValueHigh();
151  FastGPIO::Pin<A1>::setOutputValueLow();
152  _delay_us(10);
153  digitalWrite(A1, HIGH);
154  digitalWrite(A1, LOW);
155  _delay_us(10);
156 }
157 ~~~
158 
159 The pulses generated by the FastGPIO calls were 0.125&nbsp;&mu;s long because each call was compiled to just a single assembly instruction, while the pulses generated by `digitalWrite` were 5.1&nbsp;&mu;s long.

161 ## Error messages
162 
163 If you specify an invalid pin number, GCC will produce error messages like this:
164 
165 ~~~
166 warning: array subscript is above array bounds
167 warning: asm operand 0 probably doesn't match constraints
168 error: impossible constraint in 'asm'
169 ~~~
170 
171 This is because the pin number is checked at compile time and must be valid.
