8 ## Summary
9 
10 This is a library for an
11 [Arduino-compatible controller](https://www.pololu.com/arduino) that
12 interfaces with the Pololu
13 [Zumo Shield](https://www.pololu.com/catalog/product/2508),
14 [Zumo Reflectance Sensor Array](https://www.pololu.com/catalog/product/1419),
15 [Zumo robot kit](https://www.pololu.com/catalog/product/2509), and
16 [Zumo robot for Arduino](https://www.pololu.com/catalog/product/2510). It
17 provides functions to help you program an Arduino-controlled Zumo
18 robot, and the included example projects demonstrate how to use them
19 for more complex tasks.
20 
21 For more information about the library and examples, please see the
22 [Zumo Shield user's guide](https://www.pololu.com/docs/0J57).
23 
24 Please note that this library does NOT work with the Zumo 32U4 Robot, which is a very different product. The Zumo 32U4 Robot has an integrated Arduino-compatible microcontroller. If you have the Zumo 32U4 Robot, then you should not use this library and instead refer to the [Zumo 32U4 Robot documentation](https://www.pololu.com/docs/0J63).

49 ### Software
50 
51 If you are using version 1.6.2 or later of the
52 [Arduino software (IDE)](https://www.arduino.cc/en/Main/Software), you can use
53 the Library Manager to install this library:
54 
55 1. In the Arduino IDE, open the "Sketch" menu, select "Include Library", then
56  "Manage Libraries...".
57 2. Search for "ZumoShield".
58 3. Click the ZumoShield entry in the list.
59 4. Click "Install".
60 
61 If this does not work, you can manually install the library:
62 
63 1. Download the
64  [latest release archive from GitHub](https://github.com/pololu/zumo-shield-arduino-library/releases)
65  and decompress it.
66 2. Rename the folder "zumo-shield-arduino-library-xxxx" to "ZumoShield".
67 3. Drag the "ZumoShield" folder into the "libraries" directory inside your
68  Arduino sketchbook directory. You can view your sketchbook location by
69  opening the "File" menu and selecting "Preferences" in the Arduino IDE. If
70  there is not already a "libraries" folder in that location, you should make
71  the folder yourself.
72 4. After installing the library, restart the Arduino IDE.

94 ## Component libraries
95 
96 This library also includes copies of several other Arduino libraries inside it, which are used to help implement the classes and functions above.
97 
98 * [LSM303](https://github.com/pololu/lsm303-arduino)
99 * [L3G](https://github.com/pololu/l3g-arduino)
100 * [PololuBuzzer](https://github.com/pololu/pololu-buzzer-arduino)
101 * [Pushbutton](https://github.com/pololu/pushbutton-arduino)
102 * [QTRSensors](https://github.com/pololu/qtr-sensors-arduino)
103 
104 You can use these libraries in your sketch automatically without any extra installation steps and without needing to add any extra `#include` lines to your sketch.
105 
106 You should avoid adding extra `#include` lines such as `#include <Pushbutton.h>` because then the Arduino IDE might try to use the standalone Pushbutton library (if you previously installed it), and it would conflict with the copy of the Pushbutton code included in this library. The only `#include` lines needed to access all features of this library are:
107 
108 ~~~{.cpp}
109 #include <Wire.h>
110 #include <ZumoShield.h>
111 ~~~
