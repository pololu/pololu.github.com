8 ## Summary
9 
10 This is a C++ library for the Arduino IDE that helps access the on-board hardware of the [Romi 32U4 Control Board](https://www.pololu.com/product/3544).
11 
12 The Romi 32U4 Control Board turns the Romi chassis into a programmable robot based on the Arduino-compatible ATmega32U4 MCU. It has an integrated AVR ATmega32U4 microcontroller, motor drivers, encoders, buzzer, buttons, and an LSM6DS33 accelerometer and gyro.
13 
14 This library does not include code for accessing the LSM6DS33. If you want to access it, you should install the separate [LSM6 library](https://github.com/pololu/lsm6-arduino).
15 
16 This library is very similar to the [Zumo32U4](https://github.com/pololu/zumo-32u4-arduino-library) library.

18 ## Installing the library
19 
20 If you are using version 1.6.2 or later of the [Arduino software (IDE)](http://www.arduino.cc/en/Main/Software), you can use the Library Manager to install this library:
21 
22 1. In the Arduino IDE, open the "Sketch" menu, select "Include Library", then "Manage Libraries...".
23 2. Search for "Romi32U4".
24 3. Click the Romi32U4 entry in the list.
25 4. Click "Install".
26 
27 If this does not work, you can manually install the library:
28 
29 1. Download the [latest release archive from GitHub](https://github.com/pololu/romi-32u4-arduino-library) and decompress it.
30 2. Rename the folder "romi-32u4-arduino-library-master" to "Romi32U4".
31 3. Move the "Romi32U4" folder into the "libraries" directory inside your Arduino sketchbook directory. You can view your sketchbook location by opening the "File" menu and selecting "Preferences" in the Arduino IDE. If there is not already a "libraries" folder in that location, you should make the folder yourself.
32 4. After installing the library, restart the Arduino IDE.

55 ## Component libraries
56 
57 This library also includes copies of several other Arduino libraries inside it which are used to help implement the classes and functions above.
58 
59 * [FastGPIO](https://github.com/pololu/fastgpio-arduino)
60 * [PololuBuzzer](https://github.com/pololu/pololu-buzzer-arduino)
61 * [PololuHD44780](https://github.com/pololu/pololu-hd44780-arduino)
62 * [Pushbutton](https://github.com/pololu/pushbutton-arduino)
63 * [QTRSensors](https://github.com/pololu/qtr-sensors-arduino)
64 * [USBPause](https://github.com/pololu/usb-pause-arduino)
65 
66 You can use these libraries in your sketch automatically without any extra installation steps and without needing to add any extra `#include` lines to your sketch.
67 
68 You should avoid adding extra `#include` lines such as `#include <Pushbutton.h>` because then the Arduino IDE might try to use the standalone Pushbutton library (if you previously installed it), and it would conflict with the copy of the Pushbutton code included in this library. The only `#include` line needed to access all features of this library are:
69 
70 ~~~{.cpp}
71 #include <Romi32U4.h>
72 ~~~
