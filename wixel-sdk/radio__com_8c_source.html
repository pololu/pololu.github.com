00011 // For highest throughput, we want to send as much data in each packet
00012 // as possible.  But for lower latency, we sometimes need to send packets
00013 // that are NOT full.
00014 // This library will only send non-full packets if the number of packets
00015 // currently queued to be sent is small.  Specifically, that number must
00016 // not exceed TX_QUEUE_THRESHOLD.
00017 // A higher threshold means that there will be more under-populated packets
00018 // at the beginning of a data transfer (which is bad), but slightly reduces
00019 // the importance of calling radioComTxService often (which can be good).
00020 #define TX_QUEUE_THRESHOLD  1

00028 // It doesn't look at all the packets received, and it doesn't count the data that is
00029 // queued on the other Wixel.  Therefore, it is never recommended to write some kind of
00030 // program that waits for radioComRxAvailable to reach some value greater than 1: it might
00031 // never reach that value.
